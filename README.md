개발 할 때 참고 해야 하는 사항 : 

  1. 건축으로 치면 개집을 지을 때는 설계도 없이 톱질부터 해도 되지만, 사람 사는 집을 지을 때는 최소한 기둥 위치(DB 설계)는 잡아야 무너지지 않습니다. 개집 만들고 싶지 않으면 잘 하라는 뜻
  2. 구현 하려는 기능의 페이지를 간단하게 스케치라도 작성한다.
  3. 스케치 -> DB 설계 -> 구현 -> 테스트 코드 작성 및 테스트 -> API 문서 작성 -> 확인
  4. 테스트 코드 작성 안하고 체크 안하면 나중에 DB 뜯고 백엔드 뜯고 해야해서 일 두번 세번 해야하니까 그냥 편하게 만들고 테스트 하자.
  5. 하기 싫으면 좀 자거나 멍때리고 한다. 억지로 하면 더 하기 싫어짐
  6. 거창하게 만들려고 하지 말고 하나 만들고 반응 보고 덧붙혀 가는 형식으로 구현한다.
  7. 왜 이걸 쓰는지 간단한 이유라도 꼭 생각하고 넘어가야함 아무 생각 없이 하면 나중에 진짜 이 기능을 사용해야 할 때 뭐 써야할 지 감도 안 잡힘
