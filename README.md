개발 할 때 참고 해야 하는 사항 : 

  1. 건축으로 치면 개집을 지을 때는 설계도 없이 톱질부터 해도 되지만, 사람 사는 집을 지을 때는 최소한 기둥 위치(DB 설계)는 잡아야 무너지지 않습니다. (개집 만들고 싶지 않으면 잘 하라는 뜻)
  2. 구현 하려는 기능의 페이지를 간단하게 스케치라도 작성한다.
  3. 스케치 -> DB 설계 -> 구현 -> 테스트 코드 작성 및 테스트 -> API 문서 작성 -> 확인 (BEST)
  4. 테스트 코드 작성 안하고 체크 안하면 나중에 DB 뜯고 백엔드 뜯고 해야해서 일 두번 세번 해야하니까 그냥 편하게 만들고 테스트 하자.
  5. 하기 싫으면 좀 자거나 멍때리고 한다. 억지로 하면 더 하기 싫어짐 ( 폰 X , 뇌 과학적으로 폰 하면 뇌 써야해서 더 피곤하다고 함 그냥 멍때리거나 자는 게 좋다고 함)
  6. 거창하게 만들려고 하지 말고 하나 만들고 반응 보고 덧붙혀 가는 형식으로 구현한다. (실제로 크게 하려고 하면 머리 뜨거워져서 아예 내팽개침 죽어라 안 봄)
  7. 기능 개발 할 때 왜 이 기능이 필요한지 주변의 반응 보고 한다. (실제로 학원에서 센서들의 모든 값들을 각 디바이스마다 넣어서 받으려고 하였으나 주변 조언을 들어보니 그냥 메인으로 필요한 곳 한 곳만 박아두고 쓰고 위험관리 부분만 체크를 하면 유지보수성이나 여러 부분에서 좋다는 말을 듣고 깨달음)
  8. 왜 이걸 쓰는지 간단한 이유라도 꼭 생각하고 넘어가야함 아무 생각 없이 하면 나중에 진짜 이 기능을 사용해야 할 때 뭐 써야할 지 감도 안 잡 힘
